=== BUG REPORT ===

BUG-1: Метод list_to_string возвращает None вместо пустой строки
- Приоритет: High
- Локация: string_utils.py, метод list_to_string, строка 45
- Описание: При передаче пустого списка метод возвращает None,
  что противоречит ожидаемому поведению (возврат пустой строки)
- Ожидаемое поведение: Возвращать пустую строку ""
- Фактическое поведение: Возвращает None
- Воспроизведение:
  utils = StringUtils()
  result = utils.list_to_string([])  # -> None, ожидается ""

BUG-2: Нарушение PEP 8 (E501 line too long)
- Приоритет: Low
- Локация: string_utils.py, строка 5 (метод capitalize)
- Описание: Строка кода длиннее 79 символов, что нарушает стандарт PEP 8
- Рекомендация: Разбить длинную строку на несколько

BUG-3: Метод capitalize не обрабатывает None и нестроковые типы
- Приоритет: High
- Локация: string_utils.py, метод capitalize
- Описание: При передаче None или числа возникает ошибка, но не того типа,
  который ожидается в тестах. Метод должен явно проверять тип входных данных.
- Ожидаемое поведение: Выбрасывать TypeError для нестроковых типов
- Фактическое поведение: Для None - не выбрасывает исключение,
  для int - выбрасывает TypeError (вместо AttributeError)

BUG-4: Метод to_list не обрабатывает пустой разделитель
- Приоритет: Medium
- Локация: string_utils.py, метод to_list, строка 24
- Описание: При передаче delimiter="" возникает ValueError,
  так как str.split() не принимает пустой разделитель
- Ожидаемое поведение: Документировать, что delimiter не может быть пустым,
  или обрабатывать этот случай (например, возвращать список символов)
- Фактическое поведение: Выбрасывает ValueError: empty separator
- Воспроизведение:
  utils = StringUtils()
  result = utils.to_list("test", "")  # -> ValueError

BUG-5: Метод to_list не обрабатывает None
- Приоритет: High
- Локация: string_utils.py, метод to_list
- Описание: При передаче None вместо строки метод не выбрасывает исключение
- Ожидаемое поведение: Выбрасывать TypeError или AttributeError
- Фактическое поведение: None.split() вызывает AttributeError,
  но это происходит внутри, а не явно

BUG-6: Метод list_to_string не обрабатывает несписочные типы
- Приоритет: High
- Локация: string_utils.py, метод list_to_string
- Описание: При передаче None или числа вместо списка:
  * None - не вызывает исключения (возвращает None?)
  * int - вызывает TypeError, но не AttributeError
- Ожидаемое поведение: Проверять тип и выбрасывать TypeError
- Фактическое поведение: Разное поведение для разных типов

BUG-7: Несогласованность в обработке ошибок
- Приоритет: Medium
- Описание: Разные методы по-разному обрабатывают некорректные входные данные:
  * is_empty обрабатывает None корректно
  * capitalize, to_list, list_to_string - нет
- Рекомендация: Унифицировать обработку ошибок во всех методах
